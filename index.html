Project 0 — Becoming Friends with Your Camera

CS180: Intro to Computer Vision & Computational Photography

Overview

This page presents my results for Project 0. The goal is to build intuition for how perspective depends on camera position (center of projection) and how changing zoom/focal length affects the field of view but not the underlying perspective. I captured three sets of images: (1) a selfie comparison, (2) architectural perspective compression, and (3) a dolly zoom animation.

Part 1 — Selfie: Wrong vs Right

Observation

The difference in appearance is explained by perspective geometry. When the camera is close, distances from the camera to facial features vary a lot (nose ≪ ears), so perspective exaggeration makes the nose look larger. When I step back, those distance ratios shrink and the face looks flatter and more realistic. Importantly, zooming does not change perspective; it only narrows the field of view. The change in perspective comes solely from the camera position.

Part 2 — Architectural Perspective Compression

Observation

From the distant viewpoint, the background looks compressed—objects at different depths appear closer together. From the nearby wide-angle viewpoint, parallel lines diverge more and depth feels exaggerated. Again, this is driven by the change in camera position; the focal length only controls framing.

Part 3 — Dolly Zoom ("Vertigo" Shot)

Observation

Keeping the subject at a constant image size while changing the camera position forces the background to change scale: it appears to stretch or compress. This creates the classic cinematic vertigo effect and visually separates framing from perspective.

How I built the GIF (click)

I resized frames to a common size and exported an animated GIF. See the code snippet below if you want to reproduce.

Technical Notes

  • Device: smartphone camera; exposure and focus locked when possible.
  • Framing: used on-screen grid lines to match the subject size across shots.
  • Basic edits: mild JPEG compression for web (quality ~85), no filters.

Reproducibility — GIF assembly code

from PIL import Image
import glob

paths = sorted(glob.glob("images/dolly_*.jpg"))  # dolly_01.jpg ... dolly_10.jpg
frames = [Image.open(p).convert("RGB") for p in paths]

# Optionally normalize to the smallest frame size
w = min(im.width for im in frames)
h = min(im.height for im in frames)
frames = [im.resize((w, h)) for im in frames]

frames[0].save(
    "images/dolly_zoom.gif",
    save_all=True,
    append_images=frames[1:],
    duration=120,  # ms per frame
    loop=0,
    optimize=True,
)
print("Saved to images/dolly_zoom.gif")
